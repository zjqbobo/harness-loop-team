# PMO 工作流参考

> 来源：`harness-pmo` skill。记录拆解模板、优先级定义、角色原则。

## 拆解层次

```
spec/PRD
  └── 里程碑（Milestone）—— 时间盒，≤ 4 个
        └── 史诗（Epic）—— 功能主题，2-4 个/里程碑
              └── 用户故事（User Story）—— As a / I want / So that
                    └── 任务（Task）—— 仅 P0 拆解，半天~1天的粒度
```

## 优先级定义

| 优先级 | 定义 | 判断矩阵 |
|--------|------|---------|
| **P0** | 必须做 | 高影响 × 高紧迫 —— 没有它里程碑无意义 |
| **P1** | 应该做 | 高影响 × 低紧迫 或 低影响 × 高紧迫 |
| **P2** | 可以做 | 低影响 × 低紧迫 —— 做了更好，不做也能用 |

## 里程碑设计原则

1. M1 必须是 MVP 核心——没有它产品无意义
2. 最后一个里程碑是体验完善——可延后但不该缺失
3. 里程碑之间是时间先后关系，不是并列选项
4. 每个里程碑有明确的时间盒和关键交付物

## 角色原则

- **AI 提案，PM 决定** —— 每层拆解展示后等待用户确认，不自动进入下一层
- **不过度拆解** —— P1/P2 在后续里程碑迭代中细化，不提前分摊全部任务
- **相对估算** —— 用人天做量级估计，不编造精确工时
- **依赖可见** —— 每个任务标注前置依赖，让瓶颈一目了然

## 输出物

- 路径：`04-changes/<变更>/02-project-plan.md` + `02-project-plan.xlsx`
- MD：完整项目规划文档（里程碑路线图 / 史诗×故事矩阵 / 任务全景 / 统计汇总）
- Excel：格式化的多 Sheet 工作簿（项目总览 + 每个里程碑独立 Sheet），带颜色编码、冻结表头、自适应列宽
