## 二、产品概述

### 2.1 项目背景

物流业务系统中，不同国家/地区/仓库中心的业务场景需要不同的工序组合。当前系统中，每个场景的流程逻辑硬编码在 Controller 中，导致：

- 新增业务场景需要新写一套代码，开发周期长
- 调整工序顺序需要改动多处调用代码，回归风险高
- 同一工序在不同场景中无法复用，重复开发
- 业务规则散落在代码各处，运营人员无法自主配置

### 2.2 产品目标

| 维度 | 内容 |
| --- | --- |
| 核心定位 | 物流工序的配置化编排与执行引擎，实现"配置即流程" |
| 商业价值 | 新增业务场景从"开发1周"缩短为"配置10分钟"，降低运维成本 |
| 用户价值 | 运营人员可自主配置流程和规则，无需等待开发排期 |
| 成功标准（KPIs） | ① 新场景上线时间缩短 80% ② 工序节点复用率 ≥ 70% ③ 流程配置错误率 < 1% |

### 2.3 目标用户

| 角色 | 用户画像 | 核心使用场景 |
| --- | --- | --- |
| 运营人员 | 物流运营中心配置管理员 | 配置新场景的流程和节点规则，查看执行日志 |
| 开发人员 | 后端开发工程师 | 开发新的 PBC 工序节点，接入流程引擎 |
| 系统管理员 | IT 运维人员 | 监控流程执行状态，处理异常告警 |

| 本期不包含 | 节点级权限控制、流程版本管理与灰度发布、跨系统分布式事务、可视化流程编排拖拽界面 |

| 类型 | 内容 |
| --- | --- |
| 业务假设 | 每个场景的工序组合是有限的、可枚举的；节点间数据传递为同步模式 |
| 技术约束 | 基于 Spring Boot 2.7+ 生态；流程执行为单次请求链式调用，不支持异步等待 |

| 资源限制 | V1.0 仅投入 2 名后端开发，1 名测试 |

**识别共性业务，抽取通用流程，设计可变点**

![业务流程泳道图](pbc-images/prd-business-flow.png)

**流程阶段拆解**：

| 阶段 | 活动 | 任务 | 步骤 | 业务规则 | 可变点 |
| --- | --- | --- | --- | --- | --- |
| 流程匹配 | 接收业务请求 | 解析报文 | 提取国家/地区/仓库等字段 | 报文必须包含匹配条件所需字段 | 匹配条件维度可配置 |

|  | 记录执行日志 | 落库 | 记录每个节点的输入输出和耗时 | 日志保留 90 天 | 无 |

### 4.2 模块A：流程管理

#### 4.2.1 功能概述

运营人员通过配置界面管理业务流程，包括创建、编辑、删除和查看流程。流程定义了匹配条件（国家/地区/仓库中心）和有序的节点列表，实现"配置即流程"。

#### 4.2.2 前置条件 / 后置条件

| 类型 | 条件 |
| --- | --- |
| 前置条件 | 用户已登录且拥有流程配置权限；至少有一个已注册的 PBC 节点可供选择 |
| 后置条件 | 流程配置保存后立即可用于匹配和执行；删除流程不影响历史执行记录 |

#### 4.2.3 用户场景（用户故事 + 故事线法）

**故事1：新增流程**

> 作为运营人员，我想通过配置界面创建新的业务流程，以便新场景上线时无需等待开发排期，自主完成流程配置。
>
> 👉 **故事线验证**：运营小李收到通知，越南胡志明市新开了一个仓库中心HCM01，需要配置当地的标准物流流程。小李登录系统，新建流程"越南标准流程"，设置条件为国家=VN、地区=东南亚、仓库中心=HCM01，选择节点序列为[到货→分拨→发货]，保存后立即生效，无需开发介入。

**故事2：编辑流程调整工序**

> 作为运营人员，我想编辑已有流程的节点顺序和条件，以便业务规则变化时快速调整，无需改代码。
>
> 👉 **故事线验证**：华东区新增"质量抽检"工序，小李编辑"华东标准流程"，在分拨节点后插入"质量抽检"节点，保存后新的请求立即按新流程执行，历史执行记录不受影响。

**故事3：删除废弃流程**

> 作为运营人员，我想删除不再使用的流程，以便保持配置界面的整洁。
>
> 👉 **故事线验证**：华南区某个临时流程不再使用，小李选中该流程点击删除，二次确认后流程标记为删除，从列表中消失，但历史执行日志仍可查询。

#### 4.2.4 主要流程

**【流程列表页】**
1. 运营人员进入"流程管理"页面
2. 系统加载所有有效流程列表，显示名称、条件、节点数、状态、更新时间
3. 可按流程名称、国家、地区、仓库中心筛选

**【新增流程】**
1. 点击"新增流程"按钮，进入流程配置页
2. 填写流程基本信息（名称、描述）
3. 配置匹配条件：
  - 点击"添加条件"，选择维度（国家/地区/仓库中心）
  - 选择操作符（eq/ne/in/gt/lt）
  - 填写期望值（多个值用逗号分隔）
  - 可添加多个条件，选择逻辑关系（AND/OR）
4. 选择节点并排序：
  - 左侧显示所有可用节点，点击添加到右侧
  - 右侧已选节点可拖拽调整顺序
  - 每个节点可配置数据映射（上游输出字段→下游输入字段）
5. 点击"保存"→系统校验条件唯一性→保存成功，返回列表

**【编辑流程】**
1. 在流程列表点击某流程的"编辑"按钮
2. 回显该流程的所有配置
3. 运营人员修改后点击保存→校验通过→更新生效

**【删除流程】**
1. 在流程列表点击某流程的"删除"按钮
2. 弹出二次确认框："确定删除该流程吗？历史执行记录不受影响"
3. 确认后流程软删除，从列表消失

#### 4.2.5 分支流程与异常流程（七大思维方式应用）

| 场景类型 | 场景描述 | 处理方式 | 思维方式 |
| --- | --- | --- | --- |
| 分支流程 | 条件组合与已有流程冲突 | 保存前校验，提示"该条件组合已存在于流程[XXX]，请调整" | **对称思维** 正向创建→反向冲突检查 |
| 分支流程 | 选择的节点已被删除/未注册 | 节点置灰，提示"节点[XXX]未注册，请确认节点是否已部署" | **边界思维** 节点可能不存在 |
| 分支流程 | 仅选择1个节点 | 允许但提示"流程仅包含1个节点，是否确认？" | **多样性思维** 覆盖极简场景 |
| 异常流程 | 保存时网络中断 | 保留表单数据，提示"保存失败，请重试" | **异常思维** 网络异常处理 |
| 异常流程 | 同一节点被同一流程重复选择 | 不允许添加，提示"同一流程中不允许重复选择同一节点" | **边界思维** 校验数据完整性 |
| 异常流程 | 条件规则为空 | 禁用保存按钮，提示"至少配置1条匹配条件" | **边界思维** 最小值校验 |
| 异常流程 | 条件超过10条 | 禁用添加按钮，提示"最多允许10条匹配条件" | **边界思维** 最大值校验 |

### 4.3 模块B：节点管理

#### 4.3.1 功能概述

运营人员查看系统中已注册的所有 PBC 节点元数据，并配置每个节点的业务开关和外部约束规则。节点元数据由服务启动时自动扫描注册，不可手动新增或删除节点。

#### 4.3.2 前置条件 / 后置条件

| 类型 | 条件 |
| --- | --- |
| 前置条件 | 至少有一个服务已启动并注册了 PBC 节点；用户拥有节点配置权限 |
| 后置条件 | 节点配置保存后立即生效，新执行的流程使用新配置 |

#### 4.3.3 用户场景（用户故事 + 故事线法）

**故事1：查看节点列表**

> 作为运营人员，我想查看系统所有已注册的 PBC 节点，以便了解有哪些工序能力可用，便于配置流程。
>
> 👉 **故事线验证**：小李需要配置一个新流程，想知道有哪些节点可用。他进入节点列表页，看到系统自动注册的"车辆到货""中转分拨""车辆发货""质量抽检""清关"等5个节点，每个节点显示所属模块、编码、名称、描述、当前配置状态。

**故事2：配置节点约束**

> 作为运营人员，我想为某个节点配置外部约束条件，以便不满足条件的请求在进入节点时被自动拦截。
>
> 👉 **故事线验证**：小李发现"清关"节点只对海外国家生效。他进入节点配置页，为"清关"节点添加约束：国家 ne CN（不等于中国）。配置保存后，国内请求执行到清关节点时会自动被拦截跳过。

**故事3：配置业务开关**

> 作为运营人员，我想为某个节点配置业务开关，以便控制节点内部的逻辑分支，无需改代码。
>
> 👉 **故事线验证**：双11大促期间，为了提升效率，运营决定暂时跳过质量抽检的详细步骤。小李将"质量抽检"节点的"详细检验"开关关闭。保存后，执行质量抽检节点时只执行简化逻辑，不执行详细检验流程。

#### 4.3.4 主要流程

**【节点列表页】**
1. 运营人员进入"节点管理"页面
2. 系统加载所有已注册的节点列表，显示模块、节点编码、节点名称、描述、约束数、开关数、更新时间
3. 可按模块筛选节点

**【节点配置页】**
1. 在节点列表点击某节点的"配置"按钮
2. 页面分两个区域：外部约束配置、业务开关配置
3. 【外部约束配置区】
  - 显示当前已配置的约束规则列表
  - 可新增/编辑/删除约束规则
  - 每条规则：字段 + 操作符 + 值
4. 【业务开关配置区】
  - 显示当前节点支持的所有开关
  - 每个开关显示：开关名称、开关编码、当前值、开关描述
  - 可修改开关值（布尔型开关使用开关组件，枚举型使用下拉）
5. 点击"保存配置"→校验通过→保存生效

#### 4.3.5 分支流程与异常流程

| 场景类型 | 场景描述 | 处理方式 |
| --- | --- | --- |
| 分支流程 | 该节点暂无约束配置 | 显示空态提示"暂无约束配置，添加第一条约束规则" |
| 分支流程 | 约束字段名拼写错误 | 保存时校验字段名是否在节点的可用字段列表中 |
| 异常流程 | 节点元数据扫描失败 | 显示警告"节点元数据加载失败，请检查服务状态" |
| 异常流程 | 保存配置时网络中断 | 保留编辑内容，提示重试 |

### 4.4 模块C：流程执行

#### 4.4.1 功能概述

纯后端 API 模块。统一入口接收请求报文，流程引擎自动匹配流程并链式执行节点，返回聚合结果。无前端页面，供外部系统（TIS/运单系统等）调用。

#### 4.4.2 前置条件 / 后置条件

| 类型 | 条件 |
| --- | --- |
| 前置条件 | 请求方拥有 API 调用权限；至少有一个已配置且启用的流程；请求报文包含匹配条件所需字段 |
| 后置条件 | 流程执行完成；所有节点结果和执行日志已持久化；返回标准响应格式 |

#### 4.4.3 用户场景（用户故事 + 故事线法）

**故事1：正常流程执行**

> 作为外部系统（TIS），我想通过统一接口发起物流请求，以便系统自动根据请求属性匹配并执行对应的工序流程，无需关心内部流程细节。
>
> 👉 **故事线验证**：TIS系统向 /api/pbc/flow/execute 发送请求 {country:"CN", region:"华东", warehouse:"SH01", areaType:"V0055", orderId:"ORD-20260505-001"}。系统自动匹配到"华东标准流程"，依次执行：到货（校验areaType=V0055通过→状态变为ARRIVED）→ 分拨（读取上游status=ARRIVED→执行分拨逻辑→status=SORTED）→ 发货（读取上游status=SORTED→执行发货→status=DEPARTED）。全部成功后返回聚合结果，包含每个节点的状态和输出数据。

**故事2：流程匹配失败**

> 作为外部系统，我发送了一个配置中未覆盖的国家地区组合，以便得到清晰的错误提示。
>
> 👉 **故事线验证**：TIS系统发送请求 {country:"JP", region:"东亚", warehouse:"TYO01"}，没有任何流程匹配。系统返回错误码+错误信息"未找到匹配流程，请检查请求报文或联系运营配置流程"。

**故事3：节点约束校验失败中断流程**

> 作为外部系统，我发送了一个匹配流程但不满足某个节点约束的请求，以便系统在该节点中断并给出明确失败原因。
>
> 👉 **故事线验证**：TIS系统发送华东流程请求但areaType=V0066，匹配流程成功。执行到"到货"节点时，约束校验areaType=V0055失败。流程中断，返回结果success=false，失败节点code=CAR_ARR_001，错误信息"约束校验失败，字段areaType期望值V0055但实际值为V0066"，且执行日志已记录本次失败。

#### 4.4.4 主要流程

1. 外部系统携带 API Key 调用统一接口 POST /api/pbc/flow/execute
2. API Key 鉴权通过 → 继续；不通过 → 返回 401
3. 流程引擎解析请求报文
4. FlowMatcher 遍历所有启用流程的匹配条件：
  - 对每个流程，逐条匹配 conditions.rules
  - 根据 logical_operator 判断所有规则是否同时满足
5. 匹配结果判定：
  - 0个流程匹配 → 返回错误："未找到匹配流程"
  - 1个流程匹配 → 继续执行
  - ≥2个流程匹配 → 返回错误："流程配置冲突，请联系管理员"
6. FlowExecutor 创建 FlowContext，初始化请求报文
7. 按 nodes.order 排序后的节点依次执行：
  - 节点前置：ExternalRestrictions AOP 拦截，从 DB 读取该节点的约束规则
  - 约束校验：报文字段 vs 约束规则 → 不通过则节点失败，中断整个流程
  - 校验通过 → 执行节点业务逻辑：读取 configuration → 处理业务 → 返回 NodeResult
  - 将节点输出写入 FlowContext，供下游节点读取
  - 记录该节点的执行日志（输入、输出、耗时、状态）
  - 节点失败 → 快速失败机制：中断流程，返回已执行节点的结果+失败节点的错误信息
8. 所有节点执行成功 → 聚合所有 NodeResult → 返回 FlowResult
9. 记录整条流程的执行日志

#### 4.4.5 分支流程与异常流程

| 场景类型 | 场景描述 | 处理方式 |
| --- | --- | --- |
| 分支流程 | 匹配到0个流程 | 返回错误码 + "未找到匹配流程，请检查请求报文或流程配置" |
| 分支流程 | 匹配到≥2个流程 | 返回错误码 + "流程配置冲突，请联系管理员" |
| 分支流程 | 某节点无约束配置 | 跳过约束校验，直接执行业务逻辑 |
| 异常流程 | 节点约束校验失败 | 中断流程，返回失败节点信息及校验规则 |
| 异常流程 | 节点业务逻辑抛出异常 | 捕获异常，标记节点失败，中断流程，返回异常信息 |
| 异常流程 | 请求报文缺少匹配条件所需字段 | 返回错误码 + "缺少必要字段：XXX, YYY" |
| 异常流程 | 节点执行超时 | 单节点超时阈值10秒，超时则标记失败，中断流程 |
| 异常流程 | 数据库连接异常 | 返回500错误，记录告警日志 |
| 异常流程 | 1分钟内重复请求（幂等判断） | 忽略重复请求，直接返回第一次执行结果 |

#### 4.4.6 页面原型

> 此模块无前端页面，为纯后端 API 接口。
>
> 接口文档说明：
> - 接口地址：POST /api/pbc/flow/execute
> - 鉴权方式：Header 携带 X-API-Key
> - Content-Type：application/json
> - 请求体：业务报文（各字段根据流程配置决定）
> - 响应体：FlowResult 标准结构

#### 4.4.7 数据规则

**【请求报文数据规则】**

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| （动态字段） | Any | 根据流程配置决定 | 不同流程需要的字段不同，由 FlowMatcher 动态校验 |

**【响应数据规则】**

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| flow_id | Long | 匹配到的流程ID |
| flow_name | String | 流程名称 |
| success | Boolean | 整体执行结果 |
| error_message | String | 失败时的错误说明 |
| node_results | Array | 所有已执行节点的结果列表 |
| node_results[].node_code | String | 节点编码 |
| node_results[].node_name | String | 节点名称 |
| node_results[].success | Boolean | 该节点执行结果 |
| node_results[].error_message | String | 节点失败原因 |
| node_results[].data | Object | 节点输出数据 |
| node_results[].duration_ms | Long | 节点执行耗时（毫秒） |
| total_duration_ms | Long | 整体执行耗时 |

#### 4.4.8 异常与边界情况

| 异常场景 | 处理方式 |
| --- | --- |
| 请求报文为空 | 返回 400 Bad Request + "请求报文不能为空" |
| 匹配条件所需字段缺失 | 校验所有流程的所有条件字段，提示缺少哪些字段 |
| 节点执行超时 | 单节点超时阈值10秒，超时后中断线程，标记节点失败 |
| 并发执行同一流程 | 完全支持并发，每次请求创建独立的 FlowContext，无共享状态 |
| 数据库连接异常 | 返回 500 Internal Error，记录告警日志，通知管理员 |
| FlowContext 数据溢出 | 限制单个节点输出数据大小 ≤ 1MB，超过则警告但仍允许执行 |

> **（生命周期思维）** 完整覆盖从请求到响应的整个执行生命周期各环节

#### 4.4.9 验收标准（Given-When-Then）

| 序号 | 验收条件 | 优先级 |
| --- | --- | --- |
| 1 | **Given** 报文精确匹配华东标准流程且3个节点约束都满足，**When** 发起请求，**Then** 返回 success=true，且 node_results 包含3个节点的成功输出 | P0 |

| 3 | **Given** 第2个节点约束校验失败，**When** 发起请求，**Then** 返回 success=false，node_results 包含前2个节点，第2个节点标记失败 | P0 |
| 4 | **Given** 节点1输出数据包含字段status=ARRIVED，**When** 执行节点2，**Then** 节点2可从 FlowContext 读取到该字段值 | P0 |

| 7 | **Given** 有两个流程的条件都匹配报文，**When** 发起请求，**Then** 返回错误"流程配置冲突" | P0 |

#### 4.5.9 异常与边界情况

| 异常场景 | 处理方式 |
| --- | --- |
| 查询结果为空 | 显示友好空态，附带"调整筛选条件"提示 |
| 查询超时（超过5秒） | 中断查询，提示"查询超时，请缩小时间范围或稍后重试" |
| 时间跨度超过30天 | 禁用查询按钮，提示"查询时间范围不能超过30天" |
| 查看详情时数据已被清理（超过90天保留期） | 提示"该执行日志已超出90天保留期限，数据已归档，请联系管理员" |
| 导出大数量级Excel | 超过5000条数据时提示"数据量较大，可能需要较长时间，确认导出吗？" |

#### 4.5.10 验收标准（Given-When-Then）

| 序号 | 验收条件 | 优先级 |
| --- | --- | --- |
| 1 | **Given** 今天有成功和失败的执行记录，**When** 筛选状态=失败并查询，**Then** 列表只显示失败的记录 | P1 |
| 2 | **Given** 某次执行包含3个节点，第2个节点失败，**When** 查看详情，**Then** 看到3个节点的输入输出，第2个节点显示红色错误信息 | P1 |
| 3 | **Given** 筛选时间跨度超过30天，**When** 点击查询，**Then** 按钮禁用或提示"时间范围不能超过30天" | P1 |
| 4 | **Given** 查询结果共120条，**When** 翻页，**Then** 第1页50条，第2页50条，第3页20条 | P1 |
| 5 | **Given** 节点详情页展示JSON，**When** 点击JSON区域的复制按钮，**Then** 完整JSON被复制到剪贴板 | P2 |

## 五、非功能需求

### 5.1 性能需求

| 指标 | 要求 |
| --- | --- |
| 流程匹配时间 | ≤ 50ms（单请求匹配所有流程） |
| 单节点执行时间 | ≤ 3s（业务逻辑执行，不含外部依赖耗时） |
| 整体流程执行时间 | ≤ 10s（3节点标准流程） |
| 并发量支持 | ≥ 100 TPS |
| 前端页面加载 | ≤ 2s（首屏） |
| 列表查询响应 | ≤ 2s（30天内数据） |
| 详情查询响应 | ≤ 500ms |
| Excel导出 | ≤ 10s（5000条数据） |

### 5.2 安全性需求

| 需求项 | 要求 |
| --- | --- |
| 数据加密 | 执行日志中的敏感字段（如客户信息、订单隐私）脱敏展示；传输全程HTTPS |
| 访问控制 | 基于RBAC的细粒度权限控制；配置操作需登录且有对应角色；API调用需API Key鉴权 |
| 防重放攻击 | 请求携带timestamp，超时10分钟的请求拒绝；相同请求1分钟内幂等处理 |
| SQL注入防护 | 使用MyBatis参数化查询，禁止字符串拼接SQL |
| XSS防护 | 前端展示内容统一转义，JSON展示区域转义HTML标签 |
| 操作审计 | 所有配置变更（新增/编辑/删除流程、修改节点配置）记录操作人、操作时间、变更前后值 |

### 5.3 兼容性需求

| 类别 | 要求 |
| --- | --- |
| 浏览器 | Chrome 90+ / Firefox 90+ / Edge 90+ / Safari 14+ |
| 屏幕分辨率 | 最小支持 1366×768，适配 1920×1080 及以上 |
| 后端兼容 | Java 8+ / Spring Boot 2.7+ |
| 数据库兼容 | MySQL 5.7 / 8.0 |

### 5.4 可用性需求

| 需求项 | 要求 |
| --- | --- |
| 系统可用率 | ≥ 99.9%（月度） |
| 故障恢复时间 | RTO ≤ 30 分钟；单点故障不影响整体服务 |
| 数据保留 | 执行日志至少保留90天，配置变更记录至少保留1年 |
| 降级策略 | 流程引擎故障时，可降级为直接调用节点接口不经过编排 |

### 5.5 可维护性需求

| 需求项 | 要求 |
| --- | --- |
| 配置化管理 | 流程、条件、节点配置均通过界面管理，无需修改代码、重启服务 |
| 节点热插拔 | 新增 PBC 节点只需部署对应服务 jar 包，启动后自动注册，无需重启流程引擎服务 |
| 日志标准化 | 业务日志按统一格式输出，含 traceId，支持全链路追踪 |
| 健康检查接口 | 提供 /health 接口，输出各组件健康状态 |

### 5.6 监控与告警需求

| 需求项 | 要求 |
| --- | --- |
| 核心指标监控 | 流程匹配成功率、节点执行成功率、平均执行耗时、接口调用量、错误率 |
| 异常告警机制 | 节点执行失败率超过 5% 持续 5 分钟时触发告警；接口错误率超过 10% 时触发告警 |
| 慢执行告警 | 单个节点执行超过 5 秒或整体流程超过 15 秒时记录慢日志并告警 |
| 监控面板 | 提供Grafana监控面板，展示核心指标趋势图 |
